# Exercises and interview questions

## Exercises
1. Derive the probability that an observation is out-of-bag.
2. Extend the scratch tree to support entropy.
3. Implement out-of-bag accuracy in the scratch forest.
4. Compare `sqrt`, `log2`, and all-feature split selection.
5. Plot validation performance versus tree count and depth.
6. Add class weighting and evaluate minority recall.
7. Compare impurity and permutation importance under correlated features.
8. Calibrate probabilities with isotonic and sigmoid calibration.
9. Replace random splitting with grouped or temporal validation.
10. Benchmark Random Forest against Extra Trees and gradient boosting.

## Interview questions
- Why does bagging reduce variance?
- Why does Random Forest randomly select features at every split?
- What happens as the number of trees approaches infinity?
- Explain OOB estimation and its limitations.
- Why is scaling generally unnecessary?
- Why can impurity feature importance be misleading?
- How do `max_depth` and `min_samples_leaf` alter bias and variance?
- How would you handle severe class imbalance?
- When would Extra Trees outperform Random Forest?
- How would you diagnose excellent random-split validation but poor production performance?
