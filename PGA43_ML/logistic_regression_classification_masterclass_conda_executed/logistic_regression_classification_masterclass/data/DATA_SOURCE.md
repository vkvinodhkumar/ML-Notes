# Data source and reproducibility

The notebook uses `bank_marketing_teaching.csv`, generated by
`scripts/generate_teaching_data.py` when the CSV is not present.

The column names and teaching narrative are inspired by the UCI Bank Marketing
dataset described in:

> Moro, S., Cortez, P., and Rita, P. (2014). A Data-Driven Approach to Predict
> the Success of Bank Telemarketing. *Decision Support Systems*, 62, 22–31.

The rows in this repository are synthetic and are **not** the original UCI
observations. This distinction is intentional: the generator makes the lesson
offline, deterministic, lightweight, and safe to redistribute. It also injects
a small amount of missingness and exact duplication so the data-audit cells have
real work to demonstrate.

The generator uses NumPy's `default_rng` with seed `43`. With the supplied
environment and an unchanged generator, the same input CSV is recreated on the
first notebook run.

## Optional official-data exercise

Students may replace the generated CSV with the official UCI `bank-full.csv`
after reviewing its licence and schema. They should document the source,
re-run every cell, and compare which conclusions remain stable. The main
notebook does not download external data automatically.
